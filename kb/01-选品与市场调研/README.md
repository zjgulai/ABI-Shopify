# 🔍 选品与市场调研

> **节点定位**:用数据验证“做什么产品”,把直觉变成可下注的需求假设。

## 1. 本节点要解决什么
基于工具与数据完成需求验证、竞品拆解与定价测算,输出可落地的选品清单与上架优先级。

## 2. 核心子主题
- 需求验证与趋势研判(Google Trends / Perplexity)
- 选品工具与数据(Helium 10 / Jungle Scout)
- 竞品分析与差异化机会(ChatGPT / Claude)
- 定价、毛利与单位经济测算
- 关键词与搜索意图调研
- 选品评分卡与上架优先级

## 3. 真实业务细节(来源:数字化中心周报)
<!-- enrich:周报 -->
**竞品与价格带洞察**:自建「星图-市场-竞品洞察」看板,支持价格带分布、起始价格与跨度调节,做竞品价格带分析与异常数据跟进〔周报01·数据平台组/软件设计组·2026-03〕;产出《跨境电商交付后 SaaS 竞品分析报告》,对标 10+ 头部竞品、划分 4 大赛道,拆解核心功能/商业模式/生态壁垒/AI 能力以提炼蓝海切入点〔周报01·林泽宇/办公与效率组·2026-03〕。

**市占与市场份额**:亚马逊市占获取(澳大利亚、中东等区域),市占数据由 sorftime 文件入库,按 ASIN/品牌粒度计算,新增品类搜索与吸奶器市占趋势图;并接入 semrush 做 SEO+GEO 市场份额〔周报03·数据基建组/智能算法组/应用开发组·2026-05/06〕。

**需求与声量调研**:VOC 专项-Reddit 平台博文与评论数据爬取入库,作为需求与口碑信号来源〔周报01/03·数据仓库组〕;DSP 网盘/官方账号取数(Campaign/Product),新增欧洲 DSP 账号 API 对接〔周报03·获客增长组·2026-06〕;竞品动态 AI 监测〔周报03·营销AI组〕。

> **对自动化的启示**:选品决策已建立在「竞品价格带 + 市占趋势 + VOC 声量」三类数据之上,可直接作为 AI 选品 Agent 的输入特征。

## 4. 发力点(来源:Twitter 书签)
<!-- enrich:twitter -->
**把选品建成「数据层 + 信息雷达 + 多视角分析」三件套**:
- **数据层**:有作者把散落各处的原始数据整合成「AI 能直接调用的数据层」(a-stock-data),并做「多流派辩论室」分析引擎(UZI-Skill,模拟 9 大投资流派)〔Twitter @_zheergen〕。→ 迁移:把竞品价格带/市占/VOC(见 §3)统一成可被 Agent 调用的「选品数据层」,再用「多角色分析 Agent」(成本/差异化/合规)产出选品报告。
- **信息雷达**:Horizon 聚合 HN/Reddit/Twitter/GitHub,打分去重、中英双语简报、推送飞书/邮箱〔Twitter @iluciddreaming〕。→ 做「选品/趋势雷达」,每日自动简报。
- **用户画像**:从 Reddit 评论提炼用户画像〔Twitter @MindOS_Lisa〕,与周报「Reddit VOC 爬取」呼应。

## 5. Shopify 官方能力 & AI 工具
<!-- enrich:shopify官方 -->
**Shopify Catalog 让 AI 跨全网搜品**:Shopify 把 Catalog 开放给开发者(MCP/REST API),AI Agent 可检索全平台数十亿商品〔Shopify 官方〕。
> 意义:可借 Catalog 做「AI 辅助选品与定价」的市场面;也要意识到你的商品会被竞品的 AI 选品工具看到。


> **【2026 深化】** Catalog API 把全网数十亿商品结构化、供 AI Agent 发现;选品/商品数据要按 Catalog 标准做,才能「被 AI 渠道推荐」。详见 [代理式商务技术栈](../02-建站与基础设施/Shopify代理式商务技术栈2026.md)。

## 6. SOP 与自动化要点
流程:数据层采集 → 多视角分析 → 选品评分卡 → 人审定档。**可自动化**:数据采集、竞品/市占/VOC 简报、初筛评分(Agent);**人审**:终审选品。
