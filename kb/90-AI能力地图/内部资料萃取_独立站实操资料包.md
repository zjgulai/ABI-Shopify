## 2. 统一增长模型

这批资料的共同主线不是“多学几个渠道”,而是把独立站经营拆成一条可复制的增长系统:

`赛道/产品假设 -> 社媒前置信号 -> 产品力与差异化 -> Listing/PDP 承接 -> Meta/Google/Reddit/KOL/Deal 组合引流 -> GA/UTM/广告报告归因 -> 内容与客户资产沉淀 -> 复购和再投放`

核心判断:
- 独立站不是只建一个 Shopify 店,而是通过 `AOV / CVR / LTV / CAC` 管利润闭环。
- Amazon 与独立站不是二选一;资料强调 Amazon、独立站、Google、TikTok/Meta 的协同:Amazon 提供搜索/成交与评论资产,独立站沉淀用户、品牌与利润控制权。
- 流量不是单点投放,而是“种草-信任-搜索-转化-沉淀-复用”的组合系统。
- AI/Agent 不应只做文案生成,应嵌入选品、Listing、广告初始化、广告诊断、经营周报等重复流程,但关键预算、合规、产品方向仍保留人审。

## 4. 选品与产品力:从公开榜单后移到需求萌芽前

资料对传统选品的批评很明确:等 Amazon BSR 或工具数据已经公开后再进入,很容易进入同款扎堆和价格战。更高价值的入口是在用户完成交易前,去观察他们表达需求、抱怨、求链接和分享 DIY 方案的平台。

可复用方法:
- 信号源:Instagram Reels、Pinterest 收藏图、YouTube 评论、Reddit 讨论、TikTok 种草内容、众筹项目、竞品评论、社群问题。
- 高价值信号:反复出现的痛点、评论区求购买链接、用户自制替代方案、内容传播速度快但商业供给少、强情绪反馈、细分场景未被现有产品覆盖。
- 产品假设链路:社媒信号 -> 用户痛点 -> 竞品与价格带验证 -> 供应链/成本/合规检查 -> 小批量测试 -> Listing/广告验证。
- 产品力维度:功能价值、情绪价值、视觉锤、产品结构、生命周期、矩阵裂变和第二增长曲线。

对 ABI 的落地含义:
- `01-选品与市场调研` 不应只抓搜索量和竞品销量,要补“社媒前置信号雷达”。
- 选品 Agent 的输出不应只有 SKU 清单,还要输出“为什么现在做、用户在哪表达需求、现有供给缺口是什么、验证路径是什么”。
- 对母婴/跨境品类,产品假设进入下一步前必须同步 `91-合规与风控` 做安全、广告、平台规则预检。

## 5. Listing/PDP 内容工厂:从卖点库到素材需求

`listing优化模板.xlsx` 是这批资料中最结构化的 SOP。它把 Listing 从“写一段文案”拆成可批量生产的内容工程:

内容对象:
- 文案:标题、五点、长描、产品信息、主图文案、品牌 A+、高级 A+、主图视频文案、关联视频文案、广告视频文案。
- 图片:主图、品牌 A+、高级 A+、主图视频首图、关联视频首图、品牌广告视频首图。
- 视频:品宣、卖点、教程、红人测评、关联视频、高级 A+ 视频、品牌广告视频。

模板字段:
- 标题目标不是堆词,而是同时服务曝光、点击和转化。
- 五点和长描结构为“卖点 + 参数 + 埋词”。
- 高级 A+ 拆成 6 个模块,一个模块一个大卖点,每个模块 3-5 张轮播图;模块首图做卖点集合阐述,后续图一张讲一个小卖点。
- 主图组覆盖首图点击、参数、使用、场景、卖点、功能、外观、对比。

数据输入:
- 竞品库字段覆盖 ASIN、品牌、标题、类目、BSR、30 天销量/销售额、价格、Coupon、毛利率、FBA、评分、Q&A、上架时间、LQS、变体、卖家等。
- 评价分析把消费者画像、使用时刻、使用地点、行为、场景、未满足需求、好评、差评、购买动机拆成可转化的卖点输入。
- QA 分析抽取用户关心的功能与场景关键词。
- 关键词库字段覆盖月搜索量、月购买量、购买率、SPR、PPC 竞价、流量词类型、自然/广告排名、ABA 周排名、标题密度、供需比、点击集中度、建议竞价范围等。

对 ABI 的落地含义:
- Listing Agent 应按“竞品库 -> 评价/QA/VOC -> 卖点库 -> 关键词库 -> 标题/五点/图片/视频需求”的顺序工作。
- 图片/视频 Agent 的输入不能只有一句提示词,应继承卖点排序、参数、场景、放置文案、尺寸和风格要求。
- `03 Listing`、`04 素材`、`06 CRO`、`07 数据` 应共享同一套商品事实和卖点库,避免广告素材与 PDP 承诺不一致。

## 6. 全域站外流量:从投放渠道升级为资产组合

资料把站外增长拆成多个互补渠道:

- FB 群组:适合低成本自然流量,但核心是账号环境隔离、真实养号、群组规则、内容节奏和软引导。
- Slickdeals:适合价格敏感、评论基础好、折扣真实的泛需求产品;价值在短期引流、转化和排名推动,风险在虚假折扣、刷投票、过度发帖、跳转/联盟链接不合规。
- KOL/KOC/UGC:不只看粉丝量,要看垂直度、粉丝画像、内容质量和复用价值;合作产物要能进入广告、官网、EDM、SEO、Amazon A+。
- ACC/Affiliate:适合新品和低门槛成交型合作,更接近按成交付费的长期渠道资产。
- Reddit:用户高决策力但反广告,适合测评型、痛点讨论型、免费资源型内容;目标不是硬广,而是社区讨论、Google 长尾和品牌搜索资产。
- Meta/Google/CPC:冷启动用于快速测试,成长期用于扩量,成熟期与 EDM/社媒/Reddit 共同做复购和搜索资产沉淀。

对 ABI 的落地含义:
- `05-营销与引流` 的主模型应从“渠道清单”升级为“渠道角色 + 阶段 + 风控 + 数据回收”。
- 每次站外投放前都要产出目标、素材、落地页、UTM、库存、折扣、客服、评论承接和复盘口径。
- 自然社媒与红人内容不是免费替代广告,而是降低长期 CAC、提升品牌搜索和复用素材的资产层。

## 7. 广告与数据:先诊断原因,再调整动作

广告资料反复强调:ACOS/ROI 是结果,不能直接指导动作。成熟的优化顺序是:

`看报表 -> 识别变化 -> 判断原因 -> 制定策略 -> 执行动作 -> 复盘`

关键报表与用途:
- Placement Report:判断 Top of Search、Rest of Search、Product Page 哪个位置更适合产品;CTR 高 CVR 低时要回到价格、Review、主图和页面承接。
- Search Term Report:识别真实搜索意图,将高转化词 Exact 化,将高点击低转化词拆成“意图不准”或“Listing 承接不足”,长期无转化词进入否定。
- Search Impression Share:判断核心词是否还有放大空间,以及订单下降是否来自竞品抢量、预算不足或结构分散。
- 商机探测器/市场数据:判断市场容量、竞争结构、新玩家、细分饱和度,把广告从“怎么投”升级到“要不要投、投哪里、投多少”。
- SA/DSP 指标:SA 看 impression share、CTR、click、CVR、ROI、VCPM、CPC;DSP 看 DPVR、ACTR、DPV、ATC、Branded search、ECPDPV、ECPATC、PR、ROI。

广告结构推断:
- 搜索词数量多且波动大,可能是自动广告或广泛/词组匹配。
- 搜索词长期稳定,更像精准匹配。
- 搜索词分散且混入无关或品牌词,更像自动宽泛匹配。
- 搜索词集中在固定词根且只在首尾延展,更像词组匹配;词根顺序被打乱或中间插词,更像广泛匹配。

对 ABI 的落地含义:
- 广告诊断 Agent 不能只输出“加预算/降竞价/否词”,必须输出证据:哪个报表、哪个层级、样本是否足够、为什么这个动作成立。
- `07-数据与归因` 应把广告报告、GA/UTM、自然排名、品牌搜索、市场容量串起来,避免单报表误判。

## 8. 风控红线

这批资料给出的风控边界非常具体:

- Facebook:一号一环境,独立网络/设备/浏览器资料,稳定登录;新号先做真实浏览、点赞、评论、加组,不要直接发商业链接;矩阵账号分工但避免强关联。
- Slickdeals:不要虚假折扣、抬高 MSRP、低评论硬推、过度发帖、使用违规跳转/联盟链接、多账号投票或评论。
- Reddit:不要把社区当广告位;先理解 Subreddit 规则、Karma、角色分工和内容类型;硬广、重复链接和品牌自嗨容易封号或引发抵触。
- 独立站品类:灰黑类产品可以建站,但广告封号风险高,可能只能做 SEO;这类不能在自动化流程里默认走付费广告。
- 素材与红人:红人内容复用前必须确认授权范围;UGC、下载竞品素材和多平台搬运都要走版权/平台 ToS 校验。

对 ABI 的落地含义:
- 所有渠道 Agent 需要先执行 `91-合规与风控` 的渠道预检,再进入投放或发帖。
- 自动化系统不得内置刷票、多账号投票、绕规则跳转、伪造折扣等动作。
- “人审守门”至少覆盖预算、折扣、广告上线、外部社区发帖、医疗/功效措辞、红人授权和生产写操作。

## 9. Agent 化落地

`workflow01.png` 和站外课程中的 AI Workflow 都指向同一个组织模型:把运营团队拆成 Agent,每个 Agent 处理高重复、强模板、可校验的任务,人工做方向和风险决策。

可落地 Agent:
- 选品 Agent:社媒/竞品/VOC/关键词雷达 -> 产品假设 -> 竞品报告 -> 立项建议。
- Listing Agent:竞品库、评价、Q&A、关键词、卖点库 -> 标题/五点/长描/A+/图片/视频需求。
- 广告初始化 Agent:新品 SP/SB/SD/DSP 或 Meta/Google 结构、预算、关键词、否词、素材映射。
- 广告诊断 Agent:报表拉取 -> 原因归因 -> 动作建议 -> 预算/否词/版位调整草案。
- 经营 Agent:日/周/月销售、库存、广告、CRO、会员、舆情巡检 -> 周报/复盘/异常预警。
- 风控 Agent:平台规则、账号环境、素材授权、广告宣称、折扣真实性、生产写操作审批。

编排原则:
- 每个 Agent 都要有输入数据、可验证输出、人工审批点和回滚方式。
- 任何生产写操作都应走 `draft -> review -> approve -> execute -> audit`。
- 对没有稳定数据源的节点,先产出草稿和待补字段,不把推断写成事实。
