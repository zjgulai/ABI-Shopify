# T6 多源深挖执行队列

## 1. 当前事实层
- 已完成:YouTube 三频道标题/链接/发布时间编目,共 76 个 Video 实体;`视频深度萃取_精选.md` 已有 2 条内容级萃取。
- 已落地:离线入库工具 `kb/tools/t6_multisource_intake.py`,可把用户粘贴的字幕、帖子正文、清单整理成带 frontmatter 的草稿 Markdown。
- 未完成:其余高价值视频的逐字稿级萃取;抖音/小红书等中文平台清单和正文尚未由用户提供。
- 边界:不绕过外站抓取限制;不联网抓取;不调用 provider;不把仅标题编目写成内容级事实。

## 2. 输入格式
支持 JSON 数组或 JSONL。每条最少需要 `title`;建议字段如下:

```json
{
  "title": "Claude Code + Shopify = $130,000/Month",
  "url": "https://www.youtube.com/watch?v=...",
  "platform": "youtube",
  "channel": "Ac Hampton",
  "published": "2026-xx-xx",
  "source_type": "transcript",
  "nodes": ["02-建站与基础设施", "10-自动化编排"],
  "tools": ["Claude Code", "Shopify AI Toolkit"],
  "claims": ["收入数字需标注为营销话术或个案"],
  "transcript": "用户粘贴的转写文本..."
}
```

抖音/小红书可把 `source_type` 设为 `post_list` 或 `note`;`content` / `notes` 字段放正文、评论摘要或人工整理清单。

## 3. 执行命令
```bash
python3 kb/tools/t6_multisource_intake.py /path/to/t6_sources.jsonl --out kb/drafts/t6_multisource --index
```

生成草稿后,按 `视频深度萃取_模板与说明.md` 的 8 段结构人审补全,再把高价值结论提升到对应节点或专题文档。

## 4. 入库与重建
内容通过人审后执行:

```bash
cd kb/_build && python3 build_rag.py
cd ../_rag/kb_index && python3 cli.py build
python3 eval_retrieval.py --min-pass-rate 0.6
cd ../../_build && python3 build_site_data.py
```

若新增平台或明确来源,再把 Source/Video/Post/Concept 关系补入 `_kg/graph.json`,并检查图谱 0 悬挂。

## 6. 验收标准
- 内容级萃取必须来自字幕/正文/用户提供清单,不能只凭标题扩写。
- 每条高价值材料至少覆盖 8 段结构中的 6 段,并标注工具、节点映射和夸大风险。
- 新源进入 RAG 后,`eval_retrieval.py --min-pass-rate 0.6` 通过。
- 如写入 `_kg/graph.json`,通用自检显示 0 悬挂关系。

## 7. 红线
- 不抓取受限外站,不使用第三方下载/破解转写。
- 不把营销收入数字当可复现收益承诺。
- 不把草稿目录当正式知识库结论;草稿需人审后再提升。
