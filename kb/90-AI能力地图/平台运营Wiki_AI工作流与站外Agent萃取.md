# 平台运营 Wiki 萃取 · AI 工作流与站外 Agent

## 1. 抽取边界

本篇只迁入 AI 工作流和 Agent 化方法,不迁入 TikTok/Amazon 专属规则作为 Shopify 事实。涉及 Gemini、TikTok Creative Studio、Kalodata、Amazon PPC 等能力时,均标记为跨平台参考,执行前需核验当前工具和平台规则。

核心来源:
- E047: AI 驱动的内容广告闭环
- E049/E063/E065: TikTok/Gemini 视觉、选品和操盘工作流
- E050/E055: 广告诊断和大促广告计划
- E056/E057: AI 视觉营销
- E076/E105: 红人/联盟/站外 Agent 场景

## 2. Agent 化任务分层

平台运营 Wiki 资料可抽象成 6 类 Agent:

| Agent | 输入 | 输出 | 人审点 |
|---|---|---|---|
| 市场洞察 Agent | 评论、竞品、社媒、关键词、趋势 | 人群、痛点、卖点、机会缺口 | 品类方向、合规预检 |
| 内容素材 Agent | 卖点、场景、渠道规格、历史素材表现 | 脚本、图片 brief、视频 brief、素材标签 | 功效宣称、素材授权 |
| 红人/联盟 Agent | creator list、粉丝画像、历史合作、预算 | 红人分层、邀约话术、合作条款草案 | 佣金、合同、寄样、授权 |
| SEO/GEO Agent | 关键词、GSC、页面结构、FAQ | 首页 SEO、博客/FAQ、Reddit 回答草案 | 官方规则、社区规则 |
| 广告诊断 Agent | 广告报表、素材标签、落地页、订单 | 浪费流量、胜出素材、扩量/停投建议 | 预算、上线、否词 |
| 经营复盘 Agent | 订单、库存、渠道、客服、广告、内容 | 日/周/月复盘和异常预警 | 经营结论、生产写操作 |

## 3. AI 辅助洞察工作流

从 E098/E065 等资料抽象出的通用流程:

1. 数据获取:评论、竞品页面、社媒内容、搜索词、广告数据、订单反馈。
2. 数据清洗:去重、语言归一、字段标准化、异常值标记。
3. 聚类:按痛点、场景、动机、竞品、价格、内容主题分组。
4. 人工判断:去除伪洞察,判断需求真实性和商业价值。
5. 输出:产品假设、内容方向、卖点排序、验证路径。

在 ABI 中,该流程应接入 `01-选品与市场调研` 和 `00-战略与定位`,而不是只作为 prompt 生成器。

## 4. AI 视觉与素材标签化

AI 视觉资料给出可复用的素材生产原则:
- 先定义素材承担的漏斗层:点击、解释、信任、转化或复购。
- 给素材贴语义标签:痛点、解决方案、权威、生活方式、优惠、对比、UGC、教程。
- 将参数翻译成用户结果,避免只堆功能词。
- 每个图片/视频 brief 都要绑定目标页面区块、目标渠道和衡量指标。
- 失败素材要进入负样本库,用于后续避免重复生成。

输出字段建议:
```yaml
asset_id:
semantic_bucket:
funnel_stage:
channel:
target_audience:
core_claim:
proof_asset:
landing_page_section:
metric_to_watch:
risk_terms:
```

## 5. 广告诊断 Agent

E050 的 Amazon PPC 例子可迁移成通用广告诊断模式:

`竞品/关键词/素材/落地页 -> 流量意图判断 -> 无效流量识别 -> 否定/拆组/换承接 -> 复盘`

迁入 ABI 后,广告诊断 Agent 不应只说“加预算”或“降出价”。它必须输出:
- 哪些词、素材或受众带来点击但不转化。
- 这些流量与产品真实卖点是否错配。
- 是素材误导、关键词误导、落地页承接不足,还是价格/评论/物流问题。
- 推荐动作是停投、否词、换素材、换落地页、重建受众,还是保留观察。

## 6. 站外 Agent Workflow

站外营销资料中的 AI Workflow 可迁入为:

| 任务 | Agent 动作 | 验证方式 |
|---|---|---|
| FB/Reddit/社区内容 | 生成回答草案、披露语、风险词检查 | 人审社区规则与发帖身份 |
| YouTube/PR 红人开发 | 从关键词找到候选,提取频道/媒体指标,生成邀约 | 人审名单、联系信息和授权 |
| 联盟/Creator Store Front | 生成专属页面结构、佣金规则草案、UTM/coupon 字段 | 人审佣金、合同、页面上线 |
| SEO 内容 | 生成关键词地图、标题、FAQ、内链建议 | GSC/官方规则核验 |
| 经营周报 | 拉取渠道和订单数据,生成异常和动作清单 | 人审预算和生产动作 |

## 7. 红线

- AI 只生成草案,外部社区发帖、广告上线、预算调整、佣金规则和 Shopify 生产写入必须人工审批。
- 不绕过 Reddit、Facebook、YouTube、Affiliate、PR 或 Shopify 的平台规则。
- 不把未验证工具能力写成当前可用事实。
- 不把 AI 生成内容直接当作官方政策或法律意见。
