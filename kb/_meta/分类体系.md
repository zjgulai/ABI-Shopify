# 分类体系与命名规范

## 顶层分类主轴 = 全流程阶段
选定主轴:**建站 → 全自动运营 的全流程阶段**(00–10),横切层(90–92)贯穿所有阶段。
理由:用户诉求是“围绕网站建设到自动化运营整个流程节点展开”,流程阶段最贴合,且利于把三源知识与自动化蓝图对齐复用。

## 编号约定
- `00–10`:流程阶段,按时间/逻辑先后排序。
- `90–92`:横切层(AI能力地图 / 合规风控 / 组织SOP),不属于单一阶段。
- 文件夹名 = `编号-中文名`;每个文件夹至少含 `README.md` 作为该节点主文档,细分主题可另开 `子主题.md`。

## 标签维度(横切,写入 frontmatter.tags)
- **工具**:shopify / ga4 / klaviyo / midjourney / capcut / zapier …
- **团队**(对应周报职能):站点产品 / 数据 / 平台研发 / 营销AI / 供应链 / 运维 …
- **信息源**:写入 frontmatter.sources(周报 / twitter / shopify官方)。

## 文档内部结构(统一 7 段)
1. 本节点要解决什么 2. 核心子主题 3. 真实业务细节(周报)4. 发力点(Twitter)5. Shopify官方能力&AI工具 6. SOP与自动化要点 7. 关联节点

## 切块(chunking)约定
- 以 `## 一级小节` 为最小检索块;每块尽量自洽(含必要上下文)。
- 萃取条目尽量带来源标注:`〔来源:周报03 · 林龙/数据组 · 2026-06〕`。
