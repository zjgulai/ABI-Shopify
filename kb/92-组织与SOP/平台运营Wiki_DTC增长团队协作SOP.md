# 平台运营 Wiki 萃取 · DTC 增长团队协作 SOP

## 1. 抽取边界

本篇萃取平台运营 Wiki 中的团队协同、岗位 KPI、红人/联盟流程、大促节奏和项目复盘资料。源资料中的团队规模、绩效权重和案例数字只作为模板参考,不作为 ABI 固定组织要求。

## 2. 角色分工

建议 ABI 将 DTC 增长团队映射为以下角色:

| 角色 | 主要职责 | 交付物 |
|---|---|---|
| 市场/品牌负责人 | 赛道、定位、人群、品牌表达 | 市场洞察、品牌定位、季度目标 |
| 站点负责人 | Shopify 结构、页面、政策、支付、物流 | 站点发布清单、CRO backlog |
| 内容负责人 | 选题、脚本、素材、内容日历 | 素材库、内容排期、爆款复盘 |
| 红人/BD | KOL、PR、联盟、Creator Store Front | 候选池、邀约、授权、结算 |
| 投放负责人 | Meta/Google/TikTok 等广告测试和放量 | 广告计划、素材测试、预算复盘 |
| 数据负责人 | GA/GSC/订单/广告/渠道归因 | 周报、异常、实验结论 |
| 风控/审核 | 平台规则、素材授权、宣称、折扣、隐私 | 审批记录、风险清单、回滚方案 |

## 3. 标准协作链路

`目标拆解 -> 人群/卖点 -> 内容 brief -> 素材生产 -> 渠道投放/红人上线 -> 站内承接 -> 数据复盘 -> SOP 更新`

每一步的输入输出:

| 阶段 | 输入 | 输出 | 审核 |
|---|---|---|---|
| 目标拆解 | 年/季/月销售、预算、库存、类目 | 渠道目标、内容目标、预算上限 | 经营负责人 |
| Brief | 人群、卖点、场景、禁用词 | 内容/页面/广告 brief | 品牌+风控 |
| 生产 | 素材、脚本、页面、红人名单 | 待上线资产 | 品牌+授权 |
| 上线 | 广告、红人、SEO、EDM、社区 | UTM/链接/活动 | 投放/BD/人审 |
| 复盘 | GA/GSC/Shopify/广告/订单 | 动作建议、素材评级 | 数据负责人 |
| 沉淀 | 结果、失败样本、授权记录 | SOP、模板、负样本库 | 项目负责人 |

## 4. 红人和联盟流程

红人/联盟合作应从一次性沟通升级为流程化资产:

1. 建立 creator candidate pool。
2. 按人群匹配、内容质量、互动质量、历史合作分层。
3. 生成邀约和报价草案。
4. 寄样和排期。
5. 返稿审核,确认品牌风险和素材授权。
6. 上线专属链接、折扣码或 Creator Store Front。
7. 复盘点击、订单、退款、内容复用和二次合作。
8. 将优质红人纳入长期合作池。

## 5. 周期性运营节奏

| 周期 | 会议/动作 | 必看指标 |
|---|---|---|
| 日 | 广告异常、订单异常、库存/支付/客服异常 | spend、orders、CVR、refund、stock |
| 周 | 内容/红人/广告/SEO 周复盘 | CTR、CPA、ROAS、自然流量、creator orders |
| 月 | 渠道预算和素材胜率复盘 | CAC、AOV、LTV、素材胜率、渠道贡献 |
| 季 | 类目、品牌、组织能力复盘 | 毛利、复购、品牌搜索、内容资产、流程瓶颈 |
| 大促 | 预热/爆发/返场专项复盘 | 主题一致性、库存、预算、退款、会员转化 |

## 6. Agent 协作边界

AI/Agent 适合做:
- 评论、竞品和社媒信息整理。
- 素材 brief、红人邀约、PR pitch、SEO outline 草案。
- 数据异常发现和复盘初稿。
- SOP 模板和会议纪要转行动项。

AI/Agent 不直接做:
- 预算上线或扩量。
- 外部社区发帖。
- 红人合同和佣金最终确认。
- 站点生产写入。
- 法律/平台规则最终判断。

## 7. 入库结论

本资料包对 `92-组织与SOP` 的最大价值是把 DTC 增长从个人经验转为流程资产。后续 ABI 的自动化编排应围绕角色、输入、输出、审批、复盘和回滚组织,而不是只把单个工具串联起来。
