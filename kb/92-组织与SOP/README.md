# 🗂️ 组织与 SOP

> **节点定位**:从真实周报沉淀的“怎么干”:岗位、协作与可复用流程。

## 1. 本节点要解决什么
沉淀从数字化中心周报萃取的真实组织职能、协作流程与可复用 SOP,作为自动化设计的现实蓝本。

## 2. 核心子主题
- 组织职能与岗位分工(站点产品/数据/平台研发/营销AI/供应链/运维)
- 需求评审与上线流程(评审→联调→提测→验收→上线)
- 发布与运维 SOP(多站点发布、流水线)
- 跨系统协作(ERP/WMS/CIS/SCRM/独立站)
- 真实项目案例库(从周报萃取)

## 3. 真实业务细节(来源:数字化中心周报)
<!-- enrich:周报 -->
**组织职能(从周报可见)**:数字化中心下设站点产品组、网站/体验研发组、平台研发组、营销 AI 组/营销产品组、数据(应用/仓库/平台/基建)组、产研与供应链系统组、系统测试/质量交付组、软件运维组、财务系统组、Agent 工程组等。

**标准研发流程(高频出现)**:需求调研 → 评审 → 设计稿/PRD → 联调 → 用例编写 → 提测 → 冒烟 → 产品验收 → 上线,几乎贯穿所有条目(如「需求评审【已通过】」「测试用例执行」「产品验收通过」)。

**发布/运维 SOP**:多站点发布、分支触发环境流水线、健康检查自动上线与告警、metersphere 接口自动化平台。

**跨系统协作**:独立站 ↔ ERP/积加/金蝶/WMS/CIS/OMS/SCRM/菜鸟 的数据打通与权限管理。

> **对自动化的启示**:真实 SOP(「评审-联调-提测-验收-上线」+ 多系统协作)是设计 AI 自动化「人审兜底」与「异常回滚」的现实蓝本。
