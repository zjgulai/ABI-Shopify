# Listing 内容工厂 SOP

## 1. 适用边界

本 SOP 用于把一个产品从“商品资料和竞品数据”推进到可上架的 Shopify PDP / Amazon Listing / 素材 brief。它沉淀自 `inbox/listing优化模板.xlsx` 的结构化模板,并结合本知识库已有的 Shopify Catalog、Magic、PDP、CRO 与素材生产节点。

事实边界:
- `listing优化模板.xlsx` 包含 20 个工作表,本 SOP 抽取字段结构、工作流和样本范式;`5.关键词库` 已结构化导出为 801 行关键词资产,路径见 `kb/_sources/inbox-independent-site/structured/listing-keyword-assets/`。
- 本 SOP 不验证资料中具体 ASIN、销量、搜索量或竞价数字的外部真实性;这些数字进入生产前必须重新拉取最新数据。
- 对母婴、健康、安全、儿童相关产品,所有卖点、图文、视频脚本必须先通过 [[91-合规与风控]] 的宣称与素材授权检查。

## 2. 工厂总流程

标准链路:

`商品事实 -> 竞品库 -> 评价/VOC/QA -> 卖点库 -> 关键词库 -> 标题/五点/长描 -> 图片/A+/视频 brief -> PDP/CRO 人审 -> 上架/实验`

每一步必须保留输入来源、生成时间、责任人/Agent、审核状态和待补字段。不要让 Agent 在缺少数据时补造事实;缺口写入 `missing_fields`。

## 3. 输入字段字典

| 输入层 | 必备字段 | 用途 | 来源 |
|---|---|---|---|
| 商品事实 | SKU/SPU、品类、材质、尺寸、重量、核心参数、包装清单、认证、禁用词 | 防止卖点失真,支撑 Catalog/PDP 基础数据 | ERP/商品资料/供应链 |
| 竞品库 | ASIN/URL、品牌、标题、类目、BSR、30天销量/销售额、价格、Coupon、评分、评分数、Q&A数、上架时间、变体、卖家、LQS | 判断市场承接、价格带、主卖点、评价门槛 | `1.竞品库` |
| 卖点库 | 大卖点、小卖点、参数、使用场景、证据、排序 | 生成五点、A+、图片文案和视频脚本 | `2.卖点库` / 商品事实 |
| 竞品卖点汇总 | 主图卖点、A+卖点、出现次数、竞品汇总卖点 | 找共性卖点与差异化缺口 | `4.竞品卖点汇总分析` |
| 评价分析 | 人群、使用时刻、地点、行为、场景、未满足需求、好评、差评、购买动机 | 从用户语言反推卖点、风险和 CRO 假设 | `3.评价分析` |
| QA 分析 | 功能关键词、场景关键词、频率/占比 | 补 FAQ、风险解释和 PDP 信息层级 | `4.竞品QA分析` |
| 关键词库 | 关键词、翻译、旺淡季、月搜索量、月购买量、购买率、SPR、PPC竞价、流量词类型、自然/广告排名、供需比、点击集中度、建议竞价 | 标题、五点、Search/Shopping/SEO 埋词 | `5.关键词库` + `keyword_assets_full.csv/jsonl` |
| 作图需求 | 图片序号、尺寸、展示卖点、画面要求、备注、放置文案、参考风格 | 交付给设计/素材 Agent | `6.作图框架` / `7.作图需求` |

## 3.1 全量关键词资产入口

本轮新增正式专题 [[Listing关键词资产库_全量导出]],用于把 `5.关键词库` 的 801 行资产接入 Listing、广告和 Agent 工作流。

使用原则:
- Agent 读取 `keyword_assets_full.jsonl` 时必须保留 `keyword_asset_id`、`source_row` 和 `row_layout`。
- `traffic_word_types` 可以拆分为自然搜索词、SP广告词、视频广告词、品牌广告词、HR推荐词,但 228 行 compact metric 布局没有稳定流量词类型,不得臆造。
- `ppc_bid`、`monthly_search_volume`、`monthly_purchase_volume` 和 ASIN 字段只能做本地模板参考;上线前必须刷新平台数据。

## 4. 输出产物清单

| 输出 | 最低要求 | 人审重点 |
|---|---|---|
| 标题 | 覆盖第一/第二/第三梯队词根,含核心参数和主场景,不堆砌不可证实词 | 关键词自然度、品牌词使用、平台字符限制 |
| 五点 | 每点一个主卖点,结构为“结果/利益 + 参数/证据 + 场景/对象 + 关键词” | 参数真实性、合规措辞、重复度 |
| 长描/PDP 模块 | 覆盖痛点、解决方案、规格、使用场景、包装、FAQ、售后 | 信息层级、移动端可读性、与广告承接一致 |
| A+ / 高级 A+ | 6 个模块左右,一个模块一个大卖点,首图聚合,后续图拆小卖点 | 图片与文案是否一一对应 |
| 主图组 brief | 首图点击,附图覆盖参数、使用、场景、功能、外观、对比 | 首图合规、信息密度、移动端可读性 |
| 视频 brief | 品宣、卖点、教程、红人测评、关联视频、广告视频 | 授权、节奏、核心卖点一致性 |
| FAQ/Q&A | 覆盖用户高频功能、兼容性、安全、售后、使用场景 | 是否真实解决购买阻碍 |
| SEO/结构化数据 | Meta title/description、产品 schema、ALT、集合页关键词 | Catalog/AI 渠道可读性 |
| CRO 假设 | 基于评价/QA/广告报告提出 PDP 优化假设 | 样本量和验证方式 |

## 5. 标题与五点规则

标题生成规则:
- 先确定 `主词根 -> 参数/卖点 -> 次词根 -> 人群/场景 -> 适用产品` 的顺序。
- 优先吃下高意图词,但不得牺牲可读性。
- 参数必须来自商品事实或供应链确认,不能从竞品标题照搬。
- Shopify PDP 标题可以更品牌化;Amazon 标题更重关键词覆盖;两者不能无差别复用。

五点生成规则:
- 每个 bullet 只承载一个主问题,避免把多个不相关功能堆在一段里。
- 推荐结构:`【卖点标签】用户利益 + 关键参数 + 使用场景 + 可信证据`。
- 差评和未满足需求必须至少映射到一个五点、FAQ 或图片模块。
- “安全、耐用、适合儿童/母婴、医疗/健康效果”等高风险措辞必须引用证据并走合规审查。

## 6. 图片与视频 brief 模板

每张图/视频片段至少包含:

| 字段 | 说明 |
|---|---|
| `asset_id` | 图序/视频序号,如 `main_02`、`aplus_3_1`、`video_tutorial_01` |
| `format` | 主图、附图、A+、高级 A+、首图、教程视频、红人测评、广告视频 |
| `size` | 平台尺寸,如 2400x3000 或移动端裁切要求 |
| `primary_claim` | 只写一个主卖点 |
| `supporting_params` | 参数、证据、认证或使用结果 |
| `scene` | 用户、场景、地点、痛点触发时刻 |
| `copy_on_image` | 放置文案,短句优先 |
| `visual_direction` | 写实/合成/外模/场景/对比/爆炸图/步骤图 |
| `source_evidence` | 来自商品事实、评价、QA、竞品或关键词库的证据 |
| `risk_check` | 禁用词、授权、夸大、平台规则 |

## 7. Listing Agent 输入输出契约

输入建议使用结构化对象,避免只丢一段自然语言:

```json
{
  "product_facts": {},
  "competitors": [],
  "review_insights": [],
  "qa_insights": [],
  "selling_points": [],
  "keyword_pool": [],
  "brand_voice": "",
  "marketplace": "shopify|amazon",
  "locale": "en-US",
  "compliance_constraints": [],
  "missing_fields": []
}
```

Agent 必须输出:
- `fact_gaps`: 缺少哪些事实,影响哪些产物。
- `positioning`: 核心人群、核心场景、主卖点、差异化。
- `keyword_strategy`: 主词、次词、长尾词、禁用/暂不使用词。
- `listing_copy`: 标题、五点、长描、FAQ、SEO meta。
- `asset_briefs`: 主图、A+、视频、广告首图 brief。
- `cro_hypotheses`: 哪些页面元素需要 A/B 测试。
- `review_checklist`: 人审项、合规项、待确认参数。

禁止输出:
- 未经证据支持的性能数字、认证、功效承诺。
- 未授权红人/UGC 的复用建议。
- 与商品事实冲突的参数。
- 只给“优化一下标题/提高转化”这类不可执行建议。

## 8. 人审与验收

上线前最少通过 7 个检查:
1. 商品事实一致:标题、五点、A+、图片、视频、FAQ 参数一致。
2. 关键词自然:核心词覆盖但不牺牲可读性。
3. 卖点可证:所有数字、认证、功效、对比都有来源。
4. 风险通过:禁用词、平台规则、版权和红人授权已检查。
5. 移动端可读:图片字数、PDP 模块、首屏信息不拥挤。
6. 广告承接一致:广告素材、落地页和 PDP 承诺一致。
7. 数据可追踪:上线版本、UTM/实验 ID、审核记录可追溯。

验收证据:
- 生成产物包路径或链接。
- 人审记录。
- 上线前截图或预览链接。
- RAG/Agent 任务输入输出日志。
- 上线后 7/14/30 天数据复盘:CTR、CVR、ATC、退款/差评、自然排名或 SEO 表现。
