# 站外渠道风控 SOP

## 1. 适用边界

本 SOP 用于所有把流量从站外导入 Amazon、Shopify 独立站或品牌内容资产的动作,包括 Facebook 群组/账号矩阵、Slickdeals、Reddit、KOL/KOC/UGC、Affiliate、站外 CPC 和社媒内容复用。

事实边界:
- 本 SOP 来自 inbox 资料包的渠道打法与风控萃取,并结合本库已有的权限审计、素材版权、数据采集合规与人审守门原则。
- 本 SOP 是操作护栏,不是平台官方法律意见;平台规则、广告政策和社区规则可能变化,生产执行前必须重新核对最新政策。
- 严禁把刷票、多账号投票、伪造折扣、绕规则跳转、硬广灌水、未授权素材复用写入任何 Agent 或 SOP。

## 2. 共通预检闸

任何站外动作前先过 8 个预检:

| 预检项 | 必查问题 | 证据 |
|---|---|---|
| 产品适配 | 产品是否适合该渠道?是否有评论基础、价格吸引力、合规风险? | 商品页、评价、价格历史、合规清单 |
| Offer 真实 | 折扣是否真实?是否抬高 MSRP?优惠码是否可用? | 原价/促销价截图、coupon 测试 |
| 库存履约 | 流量爆发后库存、配送、客服、售后能否承接? | 库存截图、物流时效、客服排班 |
| 内容授权 | 红人、UGC、竞品素材、截图是否有授权或合理使用依据? | 授权记录、使用范围 |
| 链接合规 | 链接是否清晰、可打开、无违规跳转、UTM 可追踪? | 链接测试、UTM 表 |
| 账号环境 | 账号、设备、IP、浏览器资料是否稳定且不强关联? | 账号台账、环境台账 |
| 社区规则 | Subreddit/FB 群/Deal 站规则是否允许此类内容? | 规则截图、发帖历史 |
| 人审批准 | 预算、折扣、对外发帖、红人复用是否有人审记录? | 审批记录 |

预检输出必须包含:
- `go/no-go`
- `risk_level`: low / medium / high
- `required_approvals`
- `blocked_reasons`
- `rollback_plan`
- `tracking_plan`

## 3. Facebook 矩阵风控

允许动作:
- 一个账号对应稳定网络、设备和浏览器资料。
- 新账号先做真实浏览、关注、点赞、评论和加入垂直群组。
- 按角色分工:主账号发内容、辅助账号互动、KOC 人设账号分享体验、观察账号监控规则和反馈。
- 内容先价值/讨论/经验,再软性引导;链接可放评论、资料页、私信或资源文档,避免重复直链。

禁止动作:
- 多账号共用同一 IP、设备、浏览器资料或 Cookie。
- 新号直接进群发商业链接。
- 高频重复硬广、同文案群发、夸大宣称、违规关键词。
- 多账号互刷、强行顶帖、伪装真实用户做误导性推荐。

养号节奏:
1. D1-D3:完善资料,浏览 Feed,关注垂直页面,少量真实互动。
2. D4-D7:加入目标群组,阅读规则,参与非商业讨论。
3. W2:发布经验/问题/对比内容,观察互动与限制。
4. W3+:低频软植入,保留发帖、评论、限制和封禁原因台账。

暂停条件:
- 账号被限制、验证码频繁、群组管理员警告、同类内容互动明显下降。
- 发现 IP/设备关联风险。
- 用户质疑广告性质且无法清晰解释身份和利益关系。

## 4. Slickdeals 风控

适合产品:
- 泛需求、价格敏感、评论基础较好、折扣真实、有库存和售后承接。
- 不适合:低评论、差评突出、价格没有优势、限制条件复杂、售后风险高、灰黑品类。

三种路径:
- 红人/发帖人合作:成本更高,成功率和节奏更可控。
- 自行发帖:要求账号历史和真实社区参与,风险较高。
- 后台提交审核:更依赖 Deal 质量,节奏较慢。

材料清单:
- 品名、平台链接、优惠码、开始/结束时间、原售价、促销价、折扣力度、库存、配送、评论数/评分、图片、核心优势、适用地区、UTM/短链测试。

禁止动作:
- 虚假折扣、先抬价再打折、夸大 MSRP。
- 多账号投票/评论、刷热度、诱导投票。
- 违规 affiliate/redirect 链接、打不开或跳转不透明链接。
- 低质量产品硬推,或同一品牌/同一产品过度发帖。

上线监控:
- 票数、评论问题、点击、订单、转化、coupon 使用、库存、退款、排名变化。
- 评论区只做事实澄清和售后响应,不要争辩或引导刷好评。

## 5. Reddit 风控

核心原则:
- Reddit 不是广告位,它是讨论场。品牌目标是获得真实讨论、长尾搜索和口碑资产,不是短链硬推。

允许内容:
- 测评型:真实使用过程、优缺点、对比。
- 痛点讨论型:从用户问题出发,不急于放链接。
- 免费资源型:清单、指南、FAQ、模板。
- 透明身份内容:官方号或员工号必须如实说明身份。

账号矩阵:
- 官方号:公告、FAQ、售后回应。
- 员工号:专业解释和场景答疑,身份透明。
- KOC 号:真实体验,必须有授权和披露边界。
- 观察号:只做规则观察和舆情监控,不参与操控讨论。

禁止动作:
- 硬广标题、重复链接、跨 Subreddit 灌水。
- 伪装用户身份、诱导评论、批量投票。
- 不读规则直接发帖。
- 被质疑后删除证据或争辩升级。

自建 Subreddit:
- 用于品牌 FAQ、教程、UGC、KOC 内容沉淀。
- 不作为唯一宣传阵地;仍需参与外部垂类社区并尊重社区规则。

## 6. KOL / UGC / Affiliate 复用风控

合作前:
- 明确授权范围:平台、地域、期限、是否可二次剪辑、是否可用于广告投放、是否可用于 Amazon A+/Shopify PDP/EDM。
- 明确披露要求:赞助、样品置换、佣金、Affiliate 链接。
- 明确素材交付格式:原片、字幕、缩略图、口播脚本、静态图、使用说明。

复用前:
- 检查人物肖像、音乐、第三方素材、产品宣称、竞品对比和平台规则。
- 素材进入广告、PDP、邮件、SEO 前,必须记录授权 ID 和过期时间。

禁止动作:
- 未授权搬运红人或竞品素材。
- 剪掉赞助披露,或把样品置换包装成完全自然推荐。
- 把红人个人体验泛化为医学、安全或性能承诺。

## 7. 渠道风控 Agent 契约

输入:

```json
{
  "channel": "facebook|slickdeals|reddit|kol|affiliate|cpc",
  "product": {},
  "offer": {},
  "content_draft": "",
  "target_community_or_account": "",
  "links": [],
  "inventory": {},
  "authorization_records": [],
  "planned_budget": 0,
  "tracking": {}
}
```

输出:
- `decision`: go / revise / block
- `risk_level`: low / medium / high
- `required_changes`
- `approval_required`
- `evidence`
- `posting_checklist`
- `monitoring_plan`
- `rollback_plan`

Agent 只能做预检和草案建议;不能自动执行外部发帖、刷互动、改价、上广告预算或发送红人合同。

## 8. 审批与审计模板

每次站外动作保留一条审计记录:

| 字段 | 示例 |
|---|---|
| `campaign_id` | `sd-20260627-product-a` |
| `channel` | Slickdeals |
| `owner` | 负责人 |
| `product_url` | Shopify/Amazon URL |
| `offer` | 原价/促销价/coupon/时段 |
| `risk_level` | medium |
| `approval` | 批准人/时间 |
| `links_tested` | pass/fail |
| `inventory_checked` | pass/fail |
| `content_authorized` | pass/fail |
| `utm` | UTM 链接 |
| `monitoring_window` | 24h / 72h / 7d |
| `result` | sessions/orders/CVR/refund/comment issues |
| `lessons` | 可复用经验 |
