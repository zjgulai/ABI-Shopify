# 客户会员与 VOC 闭环 SOP

> 适用范围:Shopify 独立站会员、客服、评论、社区、VOC、EDM、复购召回和品牌口碑。
> 边界:客户数据、订单数据和客服内容涉及隐私;任何自动分群、召回、AI 客服升级都需要最小必要数据和人审策略。

## 1. 输入

| 输入 | 最小字段 | 说明 |
|---|---|---|
| 客户账户 | customer_id、国家、语言、订单次数、最近购买 | 避免导出直接联系方式到非必要工具 |
| 订单行为 | SKU、金额、折扣、履约状态、退款/退货 | 与 [[08-订单履约与供应链]] 对齐 |
| 客服工单 | 问题类型、严重度、处理状态、升级记录 | 高危问题单独标记 |
| 评论/VOC | 产品、渠道、情绪、痛点、证据句 | 进入选品、PDP 和素材优化 |
| 社区反馈 | Reddit/社媒主题、问题、品牌提及、链接 | 规则执行前外部核验 |
| 会员权益 | 等级、积分、优惠券、礼品卡、订阅/复购策略 | 需要利润和合规审查 |

## 2. VOC 到运营动作

### 2.1 VOC 分类

| 维度 | 标签 |
|---|---|
| 情绪 | positive、neutral、negative、urgent |
| 主题 | 质量、尺寸、使用方法、物流、客服、价格、竞品、功能请求 |
| 生命周期 | 购买前、下单后、履约中、使用中、复购期 |
| 影响面 | 单个客户、单 SKU、单市场、全站 |
| 处理级别 | 自动回复、客服处理、产品改进、合规升级、管理层复盘 |

### 2.2 闭环动作

| VOC 信号 | 进入动作 |
|---|---|
| 购买前 FAQ 高频 | 更新 PDP FAQ、Store AI 知识、客服话术 |
| 物流/库存投诉 | 同步 [[08-订单履约与供应链]] 异常队列 |
| 产品功能抱怨 | 回流 [[01-选品与市场调研]] 和 [[03-商品上架与Listing]] |
| 好评场景 | 进入 [[04-内容与素材生产]] 的素材 brief |
| 社区自然提问 | 进入 [[05-营销与引流]] 的中立回答草稿 |
| 复购机会 | 进入会员分层和 EDM/短信/社媒召回 |

## 3. 会员运营节奏

| 阶段 | 动作 | 输出 |
|---|---|---|
| 首购前 | FAQ、推荐、政策说明、客服承接 | `pre_purchase_faq` |
| 首购后 | 履约通知、使用指导、售后入口 | `onboarding_message` |
| 使用期 | 评论邀请、UGC 征集、问题回收 | `review_request` / `voc_record` |
| 复购期 | 分层权益、组合推荐、补货提醒 | `retention_segment` |
| 流失期 | 原因诊断、召回优惠、客服补救 | `winback_plan` |

分层字段建议:
- `customer_value`:首购、复购、高价值、风险流失。
- `product_interest`:品类、场景、问题点。
- `service_risk`:物流、质量、客服、退款。
- `permission`:可触达渠道和同意状态。

## 4. AI 客服与人审

AI 客服可处理:
- FAQ、政策页、订单状态说明。
- 商品基础推荐。
- 使用指导。
- 工单分类和摘要。

必须升级人工:
- 退款、拒付、赔偿、差评危机。
- 医疗/安全/儿童等敏感场景。
- 客户身份、支付、地址等敏感数据确认。
- 高情绪投诉和公开社区舆情。

## 5. 输出物

| 输出物 | 说明 |
|---|---|
| `voc_taxonomy` | VOC 分类和标签 |
| `customer_segments` | 会员分层和触达策略 |
| `faq_update_queue` | PDP/FAQ/Store AI 更新队列 |
| `service_risk_register` | 高危客服和舆情台账 |
| `retention_calendar` | 复购、召回、会员活动节奏 |
| `closed_loop_report` | VOC -> 页面/产品/履约/素材/客服 的闭环复盘 |

## 6. 验收

- 每条 VOC 都能追溯到来源、渠道、SKU、问题类型和处理状态。
- 负面 VOC 有 owner、截止时间和处理结果。
- FAQ 更新进入 PDP、政策页或 AI 客服知识后有复查记录。
- 会员分层只使用必要字段,敏感数据不进入公开素材和通用模型提示词。
- 客服高危问题不由 AI 自动承诺退款、补偿或法律/医疗判断。

## 7. 风险闸

| 风险 | 处理 |
|---|---|
| 客户隐私外泄 | 最小字段、脱敏、访问控制 |
| AI 客服越权承诺 | 退款、赔偿、敏感问题升级人工 |
| 社区讨论被当作可营销素材 | 社区规则和授权单独核验 |
| 会员优惠侵蚀利润 | 与 [[07-数据与归因]] 的利润/归因联动 |
| VOC 没有闭环 | 每周复盘 unresolved/high-impact 标签 |

## 8. 与站点作战台的连接

- Journey `已有 Shopify 店铺诊断` 读取本 SOP 的 `voc_taxonomy` 和 `service_risk_register`。
- Agent `经营复盘 Agent` 使用 `closed_loop_report` 连接客服、履约、素材和增长。
- `Execution Readiness` 中客户和订单读取仍需授权;当前只做本地方法论和模板。
