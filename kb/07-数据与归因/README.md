# 📊 数据与归因

> **节点定位**:给全局“装仪表盘”:埋点、归因、看板让自动化有数据可依。

## 1. 本节点要解决什么
建设独立站数据底座与归因体系:GA4/GTM 埋点、UTM、渠道归因、CLV 与各类看板,为决策与自动化提供数据燃料。

## 2. 核心子主题
- GA4 / GTM 事件管理与埋点治理
- 埋点现状盘点与规范
- UTM 工具与投放打标
- 渠道归因分析与归因看板
- CLV / 会员价值与绩效看板
- 数据入库与备份(如 BigQuery)、数据底表治理

## 3. 真实业务细节(来源:数字化中心周报)
<!-- enrich:周报 -->
**归因体系**:独立站看板规划-渠道归因分析模块、momcozy 流量渠道归因项目(开通 GA4/coupler 权限、完成 GA 审计)、用户流量来源标签(渠道判断规则+优先级+匿名用户归因)〔周报01/02·林龙/数据应用组、陆就寅〕。

**埋点与事件**:GA4 & GTM 事件管理需求、独立站埋点现状盘点、M5 Pro 等埋点需求〔周报03·林龙/数据组 / 周报01·柯于义〕。

**UTM 与备份**:广告 UTM 工具建设与内部部署试用、GA4 report 数据接入备份至 BigQuery(BQ)〔周报03·林龙/数据组·2026-06〕。

**看板与价值度量**:CLV 数值看板线上化、用户群体画像看板、复购看板、SEO BI 看板、各域数仓重构〔周报01·林龙、黄莉、韩芸〕。

> **对自动化的启示**:数据底座(GA4/GTM + 埋点 + UTM + 归因 + BQ + 看板)已具雏形,这是「全自动运营」可观测、可决策的前提。

## 4. 发力点(来源:Twitter 书签)
<!-- enrich:twitter -->
**把归因/VOC 做成「AI 简报雷达」**:Horizon 的「聚合 → 打分 → 去重 → 双语简报 → 多渠道推送」是现成范式〔Twitter @iluciddreaming〕。→ 在 GA4/UTM 归因(§3)之上加一层「每日自动洞察简报」。
**数据 Agent 架构参考**:OpenAI 的数据 Agent「单模型 + 仅 13 个工具,跑 1.5 EB / 9 万张表」〔Twitter @alexxubyte〕——少而精的工具集,是自助式数据分析 Agent 的设计范式。

## 5. Shopify 官方能力 & AI 工具
<!-- enrich:shopify官方 -->
**Sidekick Pulse + Magic 多步推理**:Pulse 主动在后台分析店铺数据、先于你提出建议;Magic 可跨「营销/库存/客群」分析销售下滑并给修复建议〔Shopify 官方〕。
> 发力点:在 GA4/UTM 归因(§3)之上,用 Pulse/Magic 做「自动诊断 + 行动建议」。

## 6. SOP 与自动化要点
流程:埋点规范 → 采集 → 归因 → 看板/简报。**可自动化**:GA4/UTM 采集、Pulse/Magic 诊断、每日简报;**人审**:指标口径。

## 8. Inbox 独立站实操资料包补充(2026-06-27)
<!-- enrich:inbox -->
新增广告资料强调:ACOS/ROI 是结果,不能直接指导动作。标准链路应为“看报表 → 识别变化 → 判断原因 → 制定策略”,并用 Placement、Search Term、Search Impression Share、商机探测器和市场/竞品数据交叉验证〔来源:手把手带你做广告报告分析〕。

广告结构可通过搜索词数量、波动与集中度反推:搜索词多且波动大,常见于自动或广泛/词组匹配;长期稳定更像精准匹配;词根固定且只在首尾延展更像词组匹配;词根顺序被打乱或中间插词更像广泛匹配。该规则适合沉淀为广告诊断 Agent 的解释层〔来源:推测规律总结&广告拆解逻辑〕。
