# ⚙️ 独立站全自动运营蓝图(端到端)

> 目标:用全 AI 自动化方式经营 Shopify 独立站——从一键建站到自动化运营。本文是把 [[00-战略与定位]]–[[09-客户与会员运营]] 串成一条 AI 流水线的总图。

## 一、三层编排架构

```
┌─ 编排层(大脑)  Claude / Agent 编排 + CLAUDE.md 规范 + 持久记忆(Trellis 式) + Skill 库
│                 〔范式来源:Twitter 书签〕
├─ 能力层(手脚)  Shopify MCP(Dev+Storefront) · Shopify Flow · Campaign Autopilot
│                 · 外部工具链(选品/素材/视频/配音) · 自建中台(广告/数据/会员)〔周报〕
└─ 数据层(地基)  Shopify 数据(目录/订单/库存) · GA4/BQ 归因 · VOC/舆情 · 竞品/市占
```

原则:**读操作放手交给 Agent,写操作设人审/审批闸**(沿用 Campaign Autopilot「设预算→审批上线→AI 执行」范式)。

## 三、分阶段落地路线图

**阶段 0 · 地基(可立即做)**
- 建数据层:打通 GA4/GTM/UTM 归因与看板([[07-数据与归因]]);确认 Shopify 库存同步与海外仓推单([[08-订单履约与供应链]])。
- 接 Shopify AI Toolkit + Dev/Storefront MCP,跑通「Claude Code 读店铺数据」。

**阶段 1 · 单点提效(1–2 月)**
- Listing:Magic 批量文案 + 多语言([[03-商品上架与Listing]])。
- 素材:Sidekick 出图 + 视频工作流([[04-内容与素材生产]])。
- 投放:Campaign Autopilot 接管 Meta/邮件([[05-营销与引流]]),人审预算。

**阶段 2 · 流程编排(3–4 月)**
- 用 Shopify Flow + Sidekick 把库存/告警/客服自动化([[08-订单履约与供应链]]、[[09-客户与会员运营]])。
- 用 Functions 把满赠/折扣/结账校验做成可复用 CRO 组件([[06-转化优化CRO]])。

**阶段 3 · Agent 编排(5 月+)**
- 把各节点「skill 化」,用编排层(CLAUDE.md 规范 + 记忆 + Skill 库)调度,实现「一句话开站/上新/调价」。
- 全程接入人审兜底、异常告警与回滚([[91-合规与风控]]、[[92-组织与SOP]])。

## 四、护栏(不可省)

- **权限分级 + 操作审计**:Agent 写操作(改价/上下架/退款/投放预算)必须留痕 + 审批。
- **合规闸**:母婴功效措辞、儿童安全、隐私/Cookie、素材版权、数据采集 ToS(见 [[91-合规与风控]])。
- **Agent 工程三件套**:行为规范(CLAUDE.md/AGENTS.md)+ 持久记忆 + 人审,贯穿每个节点(见 [[92-组织与SOP]])。
