# T7 测试店授权前置包

## 0. 当前事实层

- 本项目已具备 `shopify-ai-kb` 本地插件、AI-Toolkit/UCP Runbook、RAG/KG/site 和 T7 受控写审批模板。
- 当前尚未登录 Shopify 测试店,尚未读取真实店铺数据,尚未执行任何 Shopify 写操作。
- 本篇只覆盖测试店授权前置准备。创建店铺、登录账号、授权 CLI/Dev MCP、读取店铺、执行 mutation 都必须由用户现场确认。

## 2. 测试店创建决策

推荐目标:只创建或使用 development store / test store,并启用测试数据,不要连接生产店。

| 决策项 | 推荐 | 理由 |
|---|---|---|
| 店铺类型 | Development store / test store | 可验证读写链路,避免影响生产 |
| 数据 | Generate test data | 方便只读查询 products/themes/policies,也方便写入测试对象 |
| 命名 | `abi-t7-smoke-YYYYMMDD` | 可回溯,不与真实品牌店混淆 |
| 权限 | 最小必要权限 | 先只读,再对单个测试商品做低风险写入 |
| 审批 | 每次 mutation 单独审批 | 避免授权一次后扩大写入范围 |

## 3. 本地 preflight

先运行只读脚本:

```bash
python3 kb/tools/t7_test_store_preflight.py --store-domain your-dev-store.myshopify.com
```

也可以在网站 `#config` 配置与授权中心录入测试店域名,由页面生成本地 preflight 命令、`shopify auth login` 指引和审批文本。该页面不执行 Shopify 登录、不读取或写入店铺、不保存 Shopify token。

脚本只做本地检查:

- `node` / `npm` / `npx` / `git` 是否存在。
- Node.js 与 Git 版本是否达到当前官方口径。
- `kb/marketplace/.claude-plugin/marketplace.json`、T7 Runbook、站点数据等本地资产是否存在。
- 目标店铺域名格式是否像 Shopify development store。
- 当前 shell 是否设置了敏感环境变量名。脚本只输出变量名和 set/unset,不输出任何值。

脚本不做的事:

- 不执行 `shopify auth login`。
- 不调用 `npx @shopify/dev-mcp`。
- 不访问 Shopify 网络端点。
- 不读取或写入任何店铺数据。
- 不保存 token、password、private key、API key。

## 4. 授权前置清单

| 阶段 | 动作 | 证据 | 通过条件 |
|---|---|---|---|
| A | 用户确认目标店铺域名 | `store_domain` | 域名属于测试店 / development store |
| B | 运行本地 preflight | 脚本输出摘要 | local_requirements 为 pass 或仅剩人工授权项 |
| C | 用户现场登录 Shopify | 浏览器/CLI 登录过程 | Codex 不接触密码、2FA、token |
| D | 只读连接 | `shopify auth whoami` 或 Dev MCP 只读响应摘要 | 目标店为测试店 |
| E | 生成 mutation 预案 | JSON 预案 | 对象、字段、前后值、回滚方式明确 |
| F | 用户批准本次 mutation | 审批台账 | 批准文本仅覆盖本次低风险写入 |

## 5. 首个低风险写入建议

只允许在测试店选择一个测试商品,做以下任一操作:

- 给测试商品追加 tag: `ai-toolkit-controlled-write-smoke`。
- 写入一个测试 metafield,命名空间建议 `abi_t7`,key 建议 `smoke_status`。
- 修改测试商品 title 后缀,写后立即读回并按需回滚。

禁止目标:

- 生产店商品、真实订单、真实客户、库存扣减、支付、退款、履约、广告预算、真实折扣码发布。

## 6. 人审口径

进入真实读写前,用户需要给出类似文本:

```text
同意在 {store_domain} 测试店执行 T7 只读连接检查。
```

低风险写入前,用户需要给出类似文本:

```text
同意仅在 {store_domain} 测试店,对 {target} 执行本次 mutation: {planned_change};禁止生产店写入和资金动作。
```

没有上述明确授权时,只能停留在本地 preflight、预案、文档和只读准备层。

## 7. 与现有 Runbook 的关系

本篇是前置包。通过本篇的本地检查与用户授权后,再进入 [`AI-Toolkit / UCP 测试店受控写验收 Runbook`](AI-Toolkit_UCP测试店受控写验收Runbook.md) 的完整流程:只读连接、读店铺数据、mutation preview、人审批准、执行低风险写入、写后复查和回滚/保留决定。
