# 📦 订单履约与供应链

> **节点定位**:把货“送到”:库存、海外仓、物流、退款的闭环。

## 1. 本节点要解决什么
打通从下单到履约的链路:库存同步、海外仓与FBA/WFS、物流、退款退货,保障交付体验与库存准确。

## 2. 核心子主题
- 独立站库存同步(Shopify 库存取值与同步)
- 海外仓与三方仓(菜鸟海外仓 / FBA / WFS)
- 物流主数据与配送方案
- 调拨、箱规/箱唛与提货计划
- 退款 / 退货 / 售后链路
- 库存中心(CIS/WMS)对接与盘点

## 3. 真实业务细节(来源:数字化中心周报)
<!-- enrich:周报 -->
**Shopify 库存同步**:独立站库存同步需求(可发货 SKU 列表、Shopify 库存同步取值调整、前端界面与导入导出逻辑),已上线并回归验证〔周报01/02·宋斓曦/数据平台组·2026-03〕。

**Shopify × 海外仓**:Shopify 对接菜鸟(Cainiao)测试闭环、Shopify 平台退款、菜鸟海外仓功能验收上线、Shopify 自发货〔周报01·陈栅汕、黄华阳 / 周报03·胡海浪〕。

**仓配主数据**:物流主数据、调拨单优化、SKU 箱规、三方仓箱唛与提货计划自动导出推送供应商、FBA/WFS 箱唛自动推送〔周报01·黄华阳/产研与供应链系统组·2026-03〕。

**系统底座**:OMS/CIS/WMS 联动、库存中心、分级仓库改造(目标每 pcs 库存有归属)〔周报01·李洋洋、张莹、李磊〕。

> **对自动化的启示**:履约链路已实现「库存同步 + 海外仓对接 + 自动推单」的自动化雏形,是订单履约 Agent 的现成接口。

## 5. Shopify 官方能力 & AI 工具
<!-- enrich:shopify官方 -->
**Shopify Flow + Sidekick 自动化**:用自然语言描述(如「库存<10 → 发 Slack 告警并打标」)即可由 Sidekick 生成 Flow;Flow 为全付费计划内置、无需 Zapier/Make〔Shopify 官方〕。
**Storefront MCP** 的 get_order_status 支持对话式查单〔Shopify 官方〕。
> 发力点:把周报的库存同步/海外仓推单(§3)用 Flow/Sidekick 编排,异常自动告警。

## 6. SOP 与自动化要点
流程:库存同步 → 海外仓推单 → 物流 → 退款。**可自动化**:Flow 库存告警/自动推单、对话式查单;**人审**:异常与退款审批。

## 8. Inbox 独立站实操资料包补充(2026-06-27)
<!-- enrich:inbox -->
新增资料对履约的直接覆盖较少,但 Slickdeals/Deal 类渠道明确要求发帖前确认库存、配送、优惠码有效期、价格真实性和售后承接。短期站外流量爆发如果没有库存与客服准备,会反向伤害评论、转化和后续广告效率〔来源:Slickdeals使用指南〕。

产品力资料也提示:流量放大前要先过产品质量、风险排查和供应链可交付性,否则 Deal 与广告只会放大缺陷〔来源:产品力提升x爆品创新〕。
